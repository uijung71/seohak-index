# 0. 지수 정의 (Index Definition)

서학개미 100 지수(Seohak-100 Index)는 대한민국 개인 투자자의 미국 주식 보유 및 매매 데이터를 기반으로 구성된 수급 가중 주가 지수(Flow + Stock Weighted Index)이다.

- 기준 통화: USD (미국 달러) 및 KRW (원화 환산 병행 운영)

---

# 1. 지수 구분 (Phase Definition)

지수는 다음 세 단계로 구분된다.

## Phase 1A: Early Estimated Index (초기 추정 지수)

- 기간: 2020-01-06 ~ 2022-12-31
- Core (1~50위): 실제 보관금액 데이터 사용
- Satellite (51~N위): trading 데이터 기반 실제 선정
- 종목 수: 평균 65~70개 (데이터 가용성 반영)
- 실제 수급 데이터만 사용하며 추정을 최소화한다
- 모든 추정 구간은 명시적으로 공개된다

## Phase 1B: Expanded Estimated Index (확장 추정 지수)

- 기간: 2023-01-01 ~ 2025-12-31
- Core (1~50위): 실제 보관금액 데이터 사용
- Satellite (51~100위): 종목 풀(~400개) 기반 선정 + 대칭 배분법 보관금액 추정
- 종목 수: 항상 100개 유지
- 생존자 편향 방지: 해당 주 기준 상장일 필터링 적용
- 51~100위 보관금액은 추정값이며 이를 명시적으로 공개한다

## Phase 2: Live Index (실제 지수)

- 기간: 2026-01-01 이후
- 보관금액 + 순매수금액 모두 실제 데이터 사용
- 실제 보관 상위 100종목 기반
- 100% 실데이터 기반 지수
- Phase 1B와 체인 방식으로 자동 연결
- 지수 발표: 추정지수(매일 오전 8시) + 확정지수(매주 월요일 오전 9시) 이중 발표 구조
- 종목 구성 및 가중치: 매주 월요일 1회 갱신 (전주 데이터 기준)

> 세이브로 데이터 T+2 지연 특성:
> 월요일 매매/보관 발생 → 화요일 미국 결제 → 수요일 세이브로 반영
> 일간 가중치 갱신은 데이터 현실과 맞지 않으므로
> 전주 데이터를 합산하여 주간 단위로 종목과 가중치를 산출한다.

# 3. Phase 2 지수 발표 구조 (이중 발표)

세이브로 데이터는 결제일 기준 T+2 지연 데이터이다.
(월요일 매매/보관 발생 → 화요일 미국 결제 → 수요일 세이브로 반영)
일간 가중치 갱신은 이 데이터 현실과 맞지 않으므로,
종목 구성과 가중치는 전주 데이터를 기반으로 매주 월요일 1회 산출한다.

## 3.1 세이브로 T+2 지연 타임라인

| 매매 발생일 | 미국 결제일 | 세이브로 반영일 | 가중치 반영 시점 |
|-------------|------------|----------------|----------------|
| 월(T)       | 화(T+1)    | 수(T+2)        | 다음 주 월요일  |
| 화(T)       | 수(T+1)    | 목(T+2)        | 다음 주 월요일  |
| 수(T)       | 목(T+1)    | 금(T+2)        | 다음 주 월요일  |
| 목(T)       | 금(T+1)    | 다음 주 월(T+2)| 다다음 주 월요일|
| 금(T)       | 다음 주 월  | 다음 주 화(T+2)| 다다음 주 월요일|

## 3.2 추정지수 (Estimated Index)

- 발표 시각: 매일 오전 8시 (한국 시간 기준)
- 산출 방법: 당주 월요일 확정 W_i를 해당 주 내내 고정, 당일 종가 등락률만 반영
- 수식: 추정지수_D = 추정지수_(D-1) × (1 + Σ(W_i_당주월요일 × r_i_D))
- 한번 발표된 추정지수는 소급 수정하지 않는다
- 종목 편출입은 추정지수에 반영하지 않음 (당주 월요일 바스켓 그대로 사용)

## 3.3 확정지수 (Official Index)

- 발표 시각: 매주 월요일 오전 9시 (한국 시간 기준)
- 산출 방법: 전주 월~금 데이터를 기반으로 종목 구성 및 가중치 갱신
- S_i 기준: 전주 월~금 일간 보관금액 평균
- N_i 기준: 전주 월~금 순매수금액(buy-sell) 합계 (음수=0)
- 종목 편출입: 확정지수 발표 시점에 반영
- 가중치는 해당 주 내내 고정 (다음 월요일까지 변경 없음)

## 3.4 추정-확정 괴리율 모니터링

주간 괴리율 = |주간 추정지수 누적 - 월요일 확정지수| / 확정지수 × 100

| 경보 수준 | 기준           | 조치                        |
|-----------|----------------|-----------------------------|
| Level 1   | 0.5% 이상      | 로그 기록                   |
| Level 2   | 1.0% 이상      | 원인 분석 수행              |
| Level 3   | 2.0% 이상      | 사람 검토 필요, 공지 여부 판단 |

## 4.3 수집 실패 대응

재시도 정책:
- 수집 실패 시 10분 간격으로 3회 재시도
- 3회 재시도 후에도 실패 시 최종 실패로 처리

최종 실패 시 처리:
- 추정지수: 예정대로 오전 8시 발표 (당주 월요일 가중치 기반)
- 확정지수(월요일): 익주 월요일에 2주치 합산 발표
- Carry-forward 최대 허용 기간: 1일
- 1일 초과 시 지수 산출 중단 및 사용자 공지
- 모든 수집 실패: data_issues.csv에 COLLECTION_FAILURE 기록

## 4.4 미국 시장 휴장일 처리

- 미국 시장 휴장일: 등락률 r_i = 0으로 처리, 전일 지수 유지
- 추정지수 발표하되 "휴장일 지수 (변동 없음)" 표시
- 주간 평균/합계 계산 시 휴장일 제외 (데이터 있는 영업일 수만으로 계산)
- 한국 공휴일과 미국 휴장일이 겹치는 경우 ISO week 기준으로 주차 매칭

# 5. 종목 풀 관리 (Universe Management)

## 5.1 기본 풀 구성

- 기본 풀: Phase 1A/1B 지수 산출에 사용된 종목 전체 (~430종목)
- 유효 종목: 상장폐지 및 합병 제외 후 약 370종목
- 풀 식별 기준: ISIN + 티커 + 상장일 복합키 사용 (ISIN 재사용 오염 방지)

## 5.2 풀 신규 편입 기준

다음 조건 중 하나를 충족하는 신규 종목은 풀에 추가된다.
- 세이브로 주간 매매상위 50에 신규 진입 (기존 풀에 없는 ISIN 최초 등장 시점)
- 세이브로 일간 보관금액 상위 50에 신규 진입
- 풀 추가일과 지수 편입 후보 자격 취득일은 동일하다
- 신규 상장 종목에 대한 별도 관찰 기간은 적용하지 않는다
- 풀 추가 이력은 ticker_universe.csv에 추가일 포함하여 기록

## 5.3 풀 제거 기준

다음 중 하나에 해당하면 풀에서 제거한다.
- 상장폐지 (완전 상장폐지)
- 인수합병으로 인한 종목 소멸
- 티커 변경 및 스핀오프: ISIN 기준 신규 종목으로 승계 처리 후 구 종목 제거
- ETF 청산: 운용사 공지 기준
- 보관금액이 24개월 이상 연속으로 0인 종목
- 모든 제거 이력은 operation_log.csv에 기록

# 7. 종목 편출입 규칙 (Rebalancing)

## 7.1 편출입 방식: 버퍼 존 (Buffer Zone)

단순 순위 변동에 따른 잦은 교체를 방지하기 위해 버퍼 존 방식을 적용한다.

## 7.2 버퍼 존 기준

| 구분 | 조건                     | 기준 순위                           | 기간    |
|------|--------------------------|-------------------------------------|---------|
| 편입 | 후보 종목이 조건 연속 충족 | 전주 보관금액 평균 기준 상위 90위 이내 | 2주 연속 |
| 편출 | 보유 종목이 조건 연속 충족 | 전주 보관금액 평균 기준 110위 밖    | 2주 연속 |

편입/편출 조건은 독립적으로 적용되며, 동시에 발생할 수 있다.

## 7.3 편출입 적용 시점

- 조건 충족 확인 후 익주 월요일 확정지수 발표 시점부터 반영
- 편출입 공지: 조건 충족 주의 월요일 확정지수 발표와 동시에 사전 공지
- 당주 추정지수는 당주 월요일 바스켓을 그대로 사용

## 7.4 편출입 시 지수 연속성 처리

- 체인 방식으로 지수 연속성 유지 (Divisor 시스템 사용하지 않음)
- 편출 종목: 마지막 확정 W_i 제거 후 잔존 종목 W_i를 100%로 재정규화
- 편입 종목: 편입 시점 실제 보관금액 기준 W_i 산출 후 전체 재정규화
- 종목 교체 주의 R_t는 교체 전 가중치 기준으로 계산

## 7.5 동일 순위 충돌 처리 (Tie-breaking)

전주 보관금액 평균이 동일한 종목이 100위 경계에 걸리는 경우 ISIN 코드 오름차순으로 우선순위를 결정한다.

## 7.6 거래 정지 종목 처리

- 거래 정지 발생 즉시 편출 처리
- 다음 순위(101위) 종목을 즉시 편입 (버퍼 존 조건 없이 긴급 편입)
- 거래 재개 시 일반 편입 조건(버퍼 존 2주)을 충족해야 재편입 가능
- operation_log에 TRADING_HALT 이벤트로 기록

| name_en | 종목명                                            |
| buy     | 매수금액 (USD, 결제 기준)                         |
| sell    | 매도금액 (USD, 결제 기준)                         |
| net_buy | 순매수금액 (USD, buy - sell)                      |

> 세이브로 공식 안내:
> "매매거래 이후 수반되는 결제처리에 따른 내역이며, 결제시점은 매매시점과 상이합니다"
> → 세이브로 매매 데이터는 매매 시점이 아닌 결제 기준 데이터이다.

## 8.3 주간 집계 데이터

파일: data/processed/custody_weekly_avg.csv
- 전주 월~금 일간 보관금액 평균 (종목별)
- 산출 시점: 매주 월요일 오전

파일: data/processed/netbuy_weekly_sum.csv
- 전주 월~금 순매수(buy-sell) 합계 (종목별)
- 산출 시점: 매주 월요일 오전

> 주간 집계 데이터가 확정지수 및 가중치 산출의 실제 입력값이다.
> 일간 세이브로 데이터는 주간 집계의 원재료로만 사용된다.

## 8.4 가격 데이터 (이중 가격 체계)

### Observed Price (관측 가격)
- 출처: custody amount, trading price
- 용도: 보관금액 정합성 검증, 이상치 탐지
- 특징: corporate action 미반영, 시계열 연속성 미보장

### Return Price (수익률 계산용 가격)
- 출처: yfinance Adjusted Close
- 용도: 종목 수익률 계산(r_i), 지수 산출
- 수익률 계산에 Observed Price 사용 절대 금지

### 가격 이상 탐지 기준 (이중 조건)
다음 두 조건을 모두 충족할 때 이상 후보로 분류한다.
- 조건 1: abs(ReturnPrice 일간 변화율) > 30%
- 조건 2: Observed vs Return 괴리 > 20%
단일 조건만 충족 시 기록만 수행

## 8.5 환율 데이터 (FX)

- 출처: 한국은행 매매기준율
- KRW 지수 = USD 지수 × (해당일 USD/KRW 환율 / 기준일 USD/KRW 환율)
- 기준일(2020-01-06) USD/KRW 환율은 최초 산출 시 확정 후 고정

# 9. 가중치 계산 (Weighting)

## 9.1 자산 가중치 (S_i) — 주간 평균 보관금액 기준

```
S_i = 전주 일간 보관금액 평균_i / Σ(전주 일간 보관금액 평균_전체 100종목)
```

- 전주 월~금 일간 보관금액을 평균하여 산출
- 데이터가 있는 영업일 수만으로 평균 계산 (휴장일 제외, 분모 = 실제 데이터 있는 일수)
- 매주 월요일 오전에 전주 데이터 기반으로 산출
- Phase 1B Satellite: 대칭 배분법 추정값 사용, segment 컬럼에 SATELLITE_EST 표시

## 9.2 수급 가중치 (N_i) — 주간 순매수 합계 기준

```
순매수_i     = buy_i - sell_i
N_i          = 전주 순매수 합계_i / Σ(전주 순매수 양수 합계_전체)
순매수 ≤ 0   → N_i = 0 (W_i = S_i만 반영)
순매수 > 0   → N_i 산출 분모에 포함
```

- 전주 월~금 일간 순매수금액(buy - sell)을 합산하여 산출
- buy+sell 합산 방식 대신 buy-sell 순매수 방식 사용
- 이유: 순매도 집중 종목에 수급 가중치가 부여되는 문제 방지
- 매주 월요일 오전에 전주 데이터 기반으로 산출

## 9.3 최종 가중치

```
W_i = 0.7 × S_i + 0.3 × N_i
```

## 9.4 가중치 고정 원칙

- 가중치는 매주 월요일 1회 산출 후 해당 주 내내 고정
- 추정지수: 당주 월요일 확정 W_i를 화~금 내내 사용
- 확정지수: 매주 월요일 전주 데이터 기반 W_i로 갱신
- 주중 세이브로 데이터 수집은 계속하되 가중치 반영은 다음 월요일

## 9.5 비중 제한 (Cap)

Cap을 적용하지 않는다.
서학개미 100 지수는 한국 개인투자자의 실제 포트폴리오 성과를 추적하는 지수이며,
실제 수급 비중을 그대로 반영하는 것을 원칙으로 한다.

# 10. 수익률 계산 (Return Calculation)

## 10.1 종목 수익률

```
r_i(t) = (ReturnPrice_t - ReturnPrice_(t-1)) / ReturnPrice_(t-1)
```

반드시 Return Price 사용. Observed Price 사용 절대 금지.

## 10.2 지수 수익률

```
R_t = Σ(W_i × r_i)
```

## 10.3 지수 포인트 (체인 수익률 방식)

```
I_t = I_(t-1) × (1 + R_t)
```

Divisor 시스템은 사용하지 않는다.

## 10.4 Price Index 원칙

- 배당은 반영하지 않는다
- Corporate action 반영 항목: 액분, 병합, 티커변경, 인수합병
- NDX도 Price Index 기준 적용 → 비교 시 동일 기준 유지

# 11. 종목 이벤트 처리 (Corporate Actions)

Corporate action은 Return Price 기준으로 수익률 연속성을 유지한다.
모든 이벤트는 corporate_actions_reference.csv 기준으로 검증한다.

| 유형 | 이벤트       | 처리 방법 |
|------|--------------|-----------|
| A형  | 완전 상장폐지 | 마지막 유효 Return Price로 수익률 반영 후 제거. operation_log에 DELISTED 기록 |
| B형  | 인수합병     | 현금 인수: 인수가격을 마지막 Return Price로 처리 후 제거 / 주식 교환: 교환 비율로 신규 종목 amount 승계 |
| C형  | 티커 변경    | ISIN 기준 연결, 수익률 영향 없음 |
| D형  | 액분/병합    | Return Price 자동 조정 → 수익률 왜곡 없음. Observed Price는 변경하지 않음 |
| E형  | 데이터 누락  | 1~2일: Carry-forward, r_i = 0% / 3일 이상: 제거 |
| F형  | 거래 정지    | 즉시 편출, 다음 순위 종목 긴급 편입 |

# 14. 소급 수정 정책 (Restatement Policy)

## 14.1 소급 수정 기준

| 영향 범위               | 처리 방법                          |
|-------------------------|------------------------------------|

## 14.2 소급 수정 절차
1. 오류 발견 즉시 내부 보고
2. 영향 범위 및 수정 지수값 산출
3. 사용자 공지 (발생 원인, 수정 전후 지수값, 적용 일자)
4. 수정된 지수값으로 이력 데이터 업데이트
5. operation_log에 RESTATEMENT 이벤트로 기록

# 23. 핵심 원칙 (Principles)

1. 완벽한 재현보다 일관성을 우선한다
2. 모든 추정은 명시적으로 공개한다
3. 동일한 룰을 각 Phase 내 전 기간에 적용한다
4. Look-ahead bias를 절대 허용하지 않는다
5. 수익률 계산에는 반드시 Return Price만 사용한다
6. Observed Price와 Return Price를 혼용하지 않는다
7. 비중(W)은 반드시 비율(%) 단위로 계산한다
8. 지수 연속성은 체인 방식으로 유지한다. Divisor 방식과 혼용하지 않는다
9. Phase 1A는 실제 데이터 우선, 억지로 종목을 채우지 않는다
10. Phase 1B는 생존자 편향 방지를 위해 상장일 필터링을 반드시 적용한다
11. Cap을 적용하지 않는다. 실제 수급 비중을 그대로 반영한다
12. 한번 발표된 추정지수는 소급 수정하지 않는다
13. 확정지수는 매주 월요일 오전 9시에 발표하며, 1% 이상 오류 발견 시에만 소급 수정한다
14. 실제 데이터 전환 이후(Phase 2)는 공식 지수로 간주한다
15. 에이전트는 판단이 필요한 경우 작업을 멈추고 사람에게 보고한다
16. 방법론 변경은 최소 4주 전 사전 공지하고 4주 병행 산출 후 전환한다
17. 산출 근거 데이터는 5년간 보존하며 재현 가능한 형태로 유지한다
18. 세이브로 T+2 지연을 인지하고 주간 단위 합산이 데이터 성격에 부합함을 원칙으로 한다
