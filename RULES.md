# 0. 지수 정의 (Index Definition)

서학개미 100 지수(Seohak-100 Index)는 대한민국 개인 투자자의 미국 주식 보유 및 매매 데이터를 기반으로 구성된 수급 가중 주가 지수(Flow + Stock Weighted Index)이다.

- 기준 통화: USD (미국 달러) 및 KRW (원화 환산 병행 운영)

---

# 1. 지수 구분 (Phase Definition)

지수는 다음 세 단계로 구분된다.

## Phase 1A: Early Estimated Index (초기 추정 지수)

- 기간: 2020-01-06 ~ 2022-12-31
- Core (1~50위): 실제 보관금액 데이터 사용
- Satellite (51~N위): trading 데이터 기반 실제 선정
- 종목 수: 평균 65~70개 (데이터 가용성 반영)
- 실제 수급 데이터만 사용하며 추정을 최소화한다
- 모든 추정 구간은 명시적으로 공개된다

## Phase 1B: Expanded Estimated Index (확장 추정 지수)

- 기간: 2023-01-01 ~ 2025-12-31
- Core (1~50위): 실제 보관금액 데이터 사용
- Satellite (51~100위): 종목 풀(~400개) 기반 선정 + 대칭 배분법 보관금액 추정
- 종목 수: 항상 100개 유지
- 생존자 편향 방지: 해당 주 기준 상장일 필터링 적용
- 51~100위 보관금액은 추정값이며 이를 명시적으로 공개한다

## Phase 2: Live Index (실제 지수)

- 기간: 2026-01-01 이후
- 보관금액 + 순매수금액 모두 실제 데이터 사용
- 실제 보관 상위 100종목 기반
- 100% 실데이터 기반 단일 일간 지수
- Phase 1B와 체인 방식으로 자동 연결
- 지수 발표: 매일 단일 지수 발표 (추정/확정 이중 구조 폐지)

> 세이브로 데이터 T+2 지연 특성:
> 실제 매매 발생 → T+1 미국 결제 → T+2 세이브로 반영
> 오늘 발표 지수의 가중치(W_i)는 T+2 기준 실제 데이터를 사용한다.
> 주가(r_i)는 당일 실시간 데이터를 사용한다.
> 이는 세이브로 데이터의 구조적 특성이며 모든 구성 종목 데이터는
> 실제 결제 완료 기준의 공식 수치이다.

# 3. Phase 2 지수 발표 구조 (단일 일간 지수)

## 3.1 세이브로 T+2 지연 특성 공시

세이브로 데이터는 실제 매매일 기준 T+2 영업일에 반영된다.

| 매매 발생일 | 미국 결제일 | 세이브로 반영일 | 지수 반영 시점  |
|-------------|------------|----------------|----------------|
| 월(T)       | 화(T+1)    | 수(T+2)        | 수요일 지수    |
| 화(T)       | 수(T+1)    | 목(T+2)        | 목요일 지수    |
| 수(T)       | 목(T+1)    | 금(T+2)        | 금요일 지수    |
| 목(T)       | 금(T+1)    | 다음 주 월(T+2)| 다음 주 월 지수|
| 금(T)       | 다음 주 월  | 다음 주 화(T+2)| 다음 주 화 지수|

> 오늘 발표 지수의 가중치는 2영업일 전 실제 보관/매매 기준이다.
> 주가 등락률(r_i)은 당일 실시간 데이터를 사용한다.
> 이는 지수의 한계가 아닌 세이브로 데이터의 구조적 특성이며
> 공식 결제 완료 기준 수치임을 명시한다.

---

## 3.2 일간 지수 산출 방식

발표 시각: 매일 (미국 장 마감 후)

산출 방법:
- 매일 세이브로에서 당일 수신 가능한 최신 T+2 데이터로
  S_i, N_i, W_i를 갱신하여 지수 산출
- 주가(r_i): 당일 EODHD API 실시간/종가 데이터 사용

수식:
```
S_i(D) = 당일 수신 보관금액_i / Σ(당일 수신 보관금액_전체)
W_i(D) = S_i(D)  (v5.2 규칙: 보관액 비중 100% 반영)
R_D    = Σ(W_i(D) × r_i(D))
I_D    = I_(D-1) × (1 + R_D)
```

데이터 수신 실패 시:
- 전일 S_i, N_i, W_i를 Carry-forward 사용
- data_issues.csv에 COLLECTION_FAILURE 기록
- Carry-forward 최대 허용 기간: 2일
- 2일 초과 시 지수 산출 중단 및 사용자 공지

## 3.3 주간 리밸런싱 (매주 월요일)

종목 편출입은 매주 월요일에 반영한다.

- 버퍼 존 기준(7항) 충족 종목 편출입 반영

> 소급 수정 없음 (14항 소급 수정 정책 적용)

# 5. 종목 풀 관리 (Universe Management)

## 5.1 기본 풀 구성

- 기본 풀: Phase 1A/1B 지수 산출에 사용된 종목 전체 (~430종목)
- 유효 종목: 상장폐지 및 합병 제외 후 약 370종목
- 풀 식별 기준: ISIN + 티커 + 상장일 복합키 사용 (ISIN 재사용 오염 방지)

## 5.2 풀 신규 편입 기준

다음 조건 중 하나를 충족하는 신규 종목은 풀에 추가된다.
- 세이브로 일간 매매 상위 종목에 신규 진입 (기존 풀에 없는 ISIN 최초 등장 시점)
- 세이브로 일간 보관금액 상위 100에 신규 진입
- 풀 추가일과 지수 편입 후보 자격 취득일은 동일하다
- 신규 상장 종목에 대한 별도 관찰 기간은 적용하지 않는다
- 풀 추가 이력은 ticker_universe.csv에 추가일 포함하여 기록

## 5.3 풀 제거 기준

다음 중 하나에 해당하면 풀에서 제거한다.
- 상장폐지 (완전 상장폐지)
- 인수합병으로 인한 종목 소멸
- 티커 변경 및 스핀오프: ISIN 기준 신규 종목으로 승계 처리 후 구 종목 제거
- ETF 청산: 운용사 공지 기준
- 보관금액이 24개월 이상 연속으로 0인 종목
- 모든 제거 이력은 operation_log.csv에 기록

# 7. 종목 편출입 규칙 (Rebalancing)

## 7.1 편출입 방식: 버퍼 존 (Buffer Zone)

단순 순위 변동에 따른 잦은 교체를 방지하기 위해 버퍼 존 방식을 적용한다.

## 7.2 버퍼 존 기준

| 구분 | 조건                      | 기준 순위                        | 기간    |
|------|---------------------------|----------------------------------|---------|
| 편입 | 후보 종목이 조건 연속 충족 | T+2 보관금액 기준 상위 90위 이내 | 2주 연속 |
| 편출 | 보유 종목이 조건 연속 충족 | T+2 보관금액 기준 110위 밖       | 2주 연속 |

편입/편출 조건은 독립적으로 적용되며, 동시에 발생할 수 있다.

## 7.3 편출입 적용 시점

- 조건 충족 확인 후 익주 월요일부터 반영
- 편출입 공지: 조건 충족 주의 금요일 사전 공지
- 리밸런싱 당일 W_i는 교체 후 새 구성 기준으로 재정규화

## 7.4 편출입 시 지수 연속성 처리

- 체인 방식으로 지수 연속성 유지 (Divisor 시스템 사용하지 않음)
- 편출 종목: 마지막 W_i 제거 후 잔존 종목 W_i를 100%로 재정규화
- 편입 종목: 편입 시점 실제 보관금액 기준 W_i 산출 후 전체 재정규화
- 종목 교체 당일 R_t는 교체 전 가중치 기준으로 계산

## 7.5 동일 순위 충돌 처리 (Tie-breaking)

보관금액이 동일한 종목이 100위 경계에 걸리는 경우 ISIN 코드 오름차순으로 우선순위를 결정한다.

## 7.6 거래 정지 종목 처리

- 거래 정지 발생 즉시 편출 처리
- 다음 순위(101위) 종목을 즉시 편입 (버퍼 존 조건 없이 긴급 편입)
- 거래 재개 시 일반 편입 조건(버퍼 존 2주)을 충족해야 재편입 가능
- operation_log에 TRADING_HALT 이벤트로 기록

## 8.3 가격 데이터 (이중 가격 체계)

### Observed Price (관측 가격)
- 출처: custody amount, trading price
- 용도: 보관금액 정합성 검증, 이상치 탐지
- 특징: corporate action 미반영, 시계열 연속성 미보장

### Return Price (수익률 계산용 가격)
- 출처: yfinance Adjusted Close
- 용도: 종목 수익률 계산(r_i), 지수 산출
- 수익률 계산에 Observed Price 사용 절대 금지

### 가격 이상 탐지 기준 (이중 조건)
다음 두 조건을 모두 충족할 때 이상 후보로 분류한다.
- 조건 1: abs(ReturnPrice 일간 변화율) > 30%
- 조건 2: Observed vs Return 괴리 > 20%
단일 조건만 충족 시 기록만 수행

## 8.4 환율 데이터 (FX)

- 출처: 한국은행 매매기준율
- KRW 지수 = USD 지수 × (해당일 USD/KRW 환율 / 기준일 USD/KRW 환율)
- 기준일(2020-01-06) USD/KRW 환율은 최초 산출 시 확정 후 고정

# 9. 가중치 계산 (Weighting)

## 9.1 자산 가중치 (S_i) — 일간 T+2 보관금액 기준

```
S_i(D) = 당일 수신 보관금액_i / Σ(당일 수신 보관금액_전체 100종목)
```

- 매일 수신된 T+2 기준 실제 보관금액 사용
- 데이터가 없는 종목: 전일 보관금액 Carry-forward
- Phase 1B Satellite: 대칭 배분법 추정값 사용, segment 컬럼에 SATELLITE_EST 표시

## 9.2 최종 가중치

```
W_i(D) = S_i(D)
```

수급 가중치(N_i) 미적용 이유:
- 본 지수의 목적은 한국 투자자가 실제로 보유한 포트폴리오 성과를 추적하는 것
- 실제 투자자 수익률 = Σ(보유 종목별 보관금액 비율 × 해당 종목 수익률)
- 이는 정확히 S_i × r_i 이며 N_i는 실제 보유 비중에 없는 개념
- N_i 반영 시 실제 투자자 수익률과 괴리 발생
- 실제 수익률 추적 정확도 우선

## 9.3 가중치 갱신 원칙

- 가중치는 매일 T+2 실제 보관금액 데이터로 갱신
- 종목 편출입(리밸런싱)은 매주 월요일에만 반영
- 데이터 수신 실패 시 전일 W_i Carry-forward 사용

## 9.5 비중 제한 (Cap)

Cap을 적용하지 않는다.
서학개미 100 지수는 한국 개인투자자의 실제 포트폴리오 성과를 추적하는 지수이며,
실제 수급 비중을 그대로 반영하는 것을 원칙으로 한다.

# 10. 수익률 계산 (Return Calculation)

## 10.1 종목 수익률

```
r_i(D) = (ReturnPrice_D - ReturnPrice_(D-1)) / ReturnPrice_(D-1)
```

반드시 Return Price 사용. Observed Price 사용 절대 금지.

## 10.2 지수 수익률

```
R_D = Σ(W_i(D) × r_i(D))
```

## 10.3 지수 포인트 (체인 수익률 방식)

```
I_D = I_(D-1) × (1 + R_D)
```

Divisor 시스템은 사용하지 않는다.

## 10.4 Price Index 원칙

- 배당은 반영하지 않는다
- Corporate action 반영 항목: 액분, 병합, 티커변경, 인수합병
- NDX도 Price Index 기준 적용 → 비교 시 동일 기준 유지

# 11. 종목 이벤트 처리 (Corporate Actions)

Corporate action은 Return Price 기준으로 수익률 연속성을 유지한다.
모든 이벤트는 corporate_actions_reference.csv 기준으로 검증한다.

| 유형 | 이벤트       | 처리 방법 |
|------|--------------|-----------|
| A형  | 완전 상장폐지| 마지막 유효 Return Price로 수익률 반영 후 제거. operation_log에 DELISTED 기록 |
| B형  | 인수합병     | 현금 인수: 인수가격을 마지막 Return Price로 처리 후 제거 / 주식 교환: 교환 비율로 신규 종목 amount 승계 |
| C형  | 티커 변경    | ISIN 기준 연결, 수익률 영향 없음 |
| D형  | 액분/병합    | Return Price 자동 조정 → 수익률 왜곡 없음. Observed Price는 변경하지 않음 |
| E형  | 데이터 누락  | 1~2일: Carry-forward, r_i = 0% / 3일 이상: 제거 |
| F형  | 거래 정지    | 즉시 편출, 다음 순위 종목 긴급 편입 |

# 23. 핵심 원칙 (Principles)

1. 완벽한 재현보다 일관성을 우선한다
2. 모든 추정은 명시적으로 공개한다
3. 동일한 룰을 각 Phase 내 전 기간에 적용한다
4. Look-ahead bias를 절대 허용하지 않는다
5. 수익률 계산에는 반드시 Return Price만 사용한다
6. Observed Price와 Return Price를 혼용하지 않는다
7. 비중(W)은 반드시 비율(%) 단위로 계산한다
8. 지수 연속성은 체인 방식으로 유지한다. Divisor 방식과 혼용하지 않는다
9. Phase 1A는 실제 데이터 우선, 억지로 종목을 채우지 않는다
10. Phase 1B는 생존자 편향 방지를 위해 상장일 필터링을 반드시 적용한다
11. Cap을 적용하지 않는다. 실제 수급 비중을 그대로 반영한다
12. 단일 일간 지수를 원칙으로 한다. 추정/확정 이중 구조를 사용하지 않는다
13. 가중치는 보관금액 비율(S_i)만 사용한다. 실제 투자자 보유 비중과의 일치가 목적이다
14. 가중치(W_i)는 매일 T+2 실제 보관금액 데이터로 갱신한다
15. 오늘 발표 지수의 가중치는 T+2 기준이며 주가는 당일 실시간임을 공시한다
16. 실제 데이터 전환 이후(Phase 2)는 공식 지수로 간주한다
17. 에이전트는 판단이 필요한 경우 작업을 멈추고 사람에게 보고한다
18. 방법론 변경은 최소 4주 전 사전 공지하고 4주 병행 산출 후 전환한다
19. 산출 근거 데이터는 5년간 보존하며 재현 가능한 형태로 유지한다
